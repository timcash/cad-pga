Gear Rotation Linkage Readme Route

How a parent motor and a child motor build a kinematic hierarchy.

This example comes from the ga_cad/src_v1 Dialtone plugin and shows the core scene-graph idea: compose a parent motion with a local child motion, then apply the result to geometry in one consistent way.

Parent Motion
$$ M_{\text{gear}}(t) = \operatorname{Rotor}(t) $$

The parent gear lives at the origin and contributes the top-level rigid motion for the whole mechanism.

Local Motion
$$ M_{\text{local}}(t) = \operatorname{Translator}(x, y)\,\operatorname{Rotor}(\sin t) $$

The arm first offsets to the gear pin, then adds a smaller local oscillation relative to that pin.

World Motion
$$ M_{\text{world}}(t) = M_{\text{gear}}(t)\,M_{\text{local}}(t) $$

This is the key scene-graph step: the child transform is evaluated in the parent frame, not directly in global coordinates.

Point Update

Every vertex is transformed by the same motor rule

$$ P_{\text{new}} = M\,P\,\widetilde{M} $$

The gear polygon and the arm polygon are both just sets of points. The only difference is which motor gets applied to each set.

Hierarchy Intuition

The child inherits motion instead of recomputing world coordinates

The arm does not need to know a separate global angle and global translation. It inherits those through the parent motor and only supplies its local offset and local oscillation.

This is the same structural idea we use in CAD assemblies, robot chains, and linkage solvers, just shown in a very small animated mechanism.

Code Mapping

What the imported plugin code is doing

  • rotor(angle) builds the parent or local rotational part.
  • translator(x, y) builds the offset from the gear center to the pin joint.
  • multiply(parent, local) composes the parent and child transforms.
  • applyMotor(motor, point) applies the rigid motion to each polygon vertex.
Why It Fits This Library

A bridge between CAD hierarchy and PGA language

This demo is intentionally lighter than the full 3D ganja.js examples. It focuses on the algebraic composition idea itself, which makes it a useful stepping stone between simple rigid motion and larger CAD mechanism examples.

It also gives the library a clean 2D example with explicit pan and zoom, which complements the 3D pages.
