CAD PGA

Projective Geometric Algebra examples, with demos and notes.

This repo is a small library of examples. Each entry gets a live route for interaction and a readme route for equations, code, and interpretation. The notes are intentionally code-faithful: when a page is exact PGA, it says so; when a page is a simpler SE(2) or WoS-style bridge, it says that too.

Library pattern

One route for seeing the motion, one route for checking the math.

The shared language across the examples is still geometric: motors move rigid bodies, boundary sums encode content, and algebraic distance queries drive the meshless heat example. The repo docs in docs/OUTLINE.md keep the longer bibliography and verification notes behind those patterns.

Mesh Cleanup

The mesh example evaluates polygon area from the oriented boundary rather than from one chosen fan triangulation. The notes page now points to the verified arXiv preprint and distinguishes the varying absolute cone sum from the invariant ideal-norm area.

$$ \begin{aligned} \partial K &= \sum_i E_i \\ \Pi(o) &= o \vee \partial K \\ A(K) &= \frac{1}{2}\left\| \Pi(o) \right\|_{\infty} \end{aligned} $$ 
let sumEdges = (pts[0] & pts[1]) * 0.0;

for (let i = 0; i < pts.length; i += 1) {
  const p1 = pts[i];
  const p2 = pts[(i + 1) % pts.length];
  sumEdges = sumEdges + (p1 & p2);
}

const netAreaPlane = apex & sumEdges;
const netArea = 0.5 * netAreaPlane.Length;

CNC Kernel Simulator

The CNC example treats spindle rotation and feed translation as one motor, then applies that motor to the full endmill geometry. The notes page makes the axis line and transform order explicit, and marks the red swept block as an approximation rather than an exact Boolean sweep.

$$ \begin{aligned} \ell_z &= \mathbf{e}_{12} \\ \mathbf{R}_z(\theta) &= \cos\!\left(\frac{\theta}{2}\right) + \sin\!\left(\frac{\theta}{2}\right)\ell_z \\ \mathbf{T}_x(d_x) &= 1 - \frac{1}{2} d_x \mathbf{e}_{01} \\ \mathbf{M}(t) &= \mathbf{T}_x(d_x(t))\,\mathbf{R}_z(\theta(t)) \end{aligned} $$ 
const rotor = Math.cos(theta / 2) + Math.sin(theta / 2) * 1e12;
const translator = 1 - 0.5 * dx * 1e01;
const motor = translator * rotor;

const drillTransformed = drillBase.map((poly) =>
  poly.map((p) => motor >>> p)
);

Gear Rotation Linkage

The gear example is the lightest entry in the library. It uses a small SE(2)-style helper in code, not full ganja.js multivectors, so the notes now show the exact parent-local-world equations first and the PGA sandwich-product form second.

$$ \begin{aligned} \phi_{\text{world}} &= \phi_{\text{gear}} + \phi_{\text{local}} \\ t_{\text{world}} &= t_{\text{gear}} + R(\phi_{\text{gear}})\,t_{\text{local}} \\ p_{\text{world}} &= R(\phi_{\text{world}})\,p_{\text{local}} + t_{\text{world}} \end{aligned} $$ 
const mGear = rotor(gearAngle);
const mLocalOffset = translator(gearRadius - 0.2, 0);
const mLocalRotation = rotor(armLocalOscillation);
const mLocalArm = multiply(mLocalOffset, mLocalRotation);
const mWorldArm = multiply(mGear, mLocalArm);

Meshless FEA WoS

The route name says FEA, but the current example is closer to a meshless harmonic heat solve. PGA wall lines and a signed-distance disk feed a classic WoS-style process based on nearest-boundary distance. The notes now call out that this is not a full Walk on Stars or finite-element implementation.

$$ \begin{aligned} d(P,\partial \Omega) &= \min \left(\min_i \left| (P \wedge \widehat{L}_i)\mathbf{e}_{012} \right|, \left|\sqrt{x^2 + y^2} - r\right|\right) \\ X_{k+1} &= X_k + d_k(\cos \theta_k, \sin \theta_k) \\ u(x) &= \mathbb{E}[g(X_{\tau})] \end{aligned} $$ 
const query = getShortestDistance(curr);
const angle = Math.random() * Math.PI * 2;

curr = point(
  getX(curr) + Math.cos(angle) * query.d,
  getY(curr) + Math.sin(angle) * query.d
);

Adding examples

The structure is intentionally small enough to grow.

New entries do not need a framework. They just need a demo route, a notes route, and one short place on the homepage. That keeps the library easy to browse and easy to extend while still leaving room for source review and code-faithful MathJax on the note pages.
