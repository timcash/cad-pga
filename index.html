CAD PGA

Projective Geometric Algebra examples, with demos and notes.

This repo is a small library of examples. Each entry gets a live route for interaction and a readme route for equations, code, and interpretation. The point is to keep the demos executable and keep the math close.

Library pattern

One route for seeing the motion, one route for understanding it.

The shared language across the examples is the same: split Euclidean and ideal content cleanly, then use motors to move geometry without leaving the algebra.

$$ \begin{aligned} A &= A_E + \mathbf{e}_0 A_I \\ X' &= \mathbf{M} X \widetilde{\mathbf{M}} \\ \mathbf{M} &= \mathbf{T}\mathbf{R} \end{aligned} $$

Mesh Cleanup

The mesh example evaluates polygon area from the oriented boundary rather than from one chosen fan triangulation. Moving the gauge point changes the cone decomposition but not the signed area encoded by the boundary sum.

$$ \begin{aligned} \partial P &= \sum_i E_i \\ A(P) &= \frac{1}{2}\left\| o \vee \partial P \right\|_{\infty} \end{aligned} $$ 
let sumEdges = (pts[0] & pts[1]) * 0.0;

for (let i = 0; i < pts.length; i += 1) {
  const p1 = pts[i];
  const p2 = pts[(i + 1) % pts.length];
  sumEdges = sumEdges + (p1 & p2);
}

const netAreaPlane = apex & sumEdges;
const netArea = 0.5 * netAreaPlane.Length;

CNC Kernel Simulator

The CNC example treats spindle rotation and feed translation as one motor, then applies that motor to the full endmill geometry. The red block is a simple swept-volume stand-in for the removed material.

$$ \begin{aligned} \mathbf{R}(\theta) &= \cos\!\left(\frac{\theta}{2}\right) + \sin\!\left(\frac{\theta}{2}\right)\mathbf{e}_{12} \\ \mathbf{T}(d_x) &= 1 - \frac{1}{2} d_x \mathbf{e}_{01} \\ \mathbf{M}(t) &= \mathbf{T}(d_x(t))\,\mathbf{R}(\theta(t)) \end{aligned} $$ 
const rotor = Math.cos(theta / 2) + Math.sin(theta / 2) * 1e12;
const translator = 1 - 0.5 * dx * 1e01;
const motor = translator * rotor;

const drillTransformed = drillBase.map((poly) =>
  poly.map((p) => motor >>> p)
);

Gear Rotation Linkage

The gear example is the lightest entry in the library: a parent rotation, a local child offset and oscillation, and a composed world transform driving a small mechanism. It is a clean scene-graph example.

$$ \begin{aligned} M_{\text{gear}} &= \operatorname{Rotor}(t) \\ M_{\text{local}} &= \operatorname{Translator}(x, y)\,\operatorname{Rotor}(\sin t) \\ M_{\text{world}} &= M_{\text{gear}}\,M_{\text{local}} \end{aligned} $$ 
const mGear = rotor(gearAngle);
const mLocalOffset = translator(gearRadius - 0.2, 0);
const mLocalRotation = rotor(armLocalOscillation);
const mLocalArm = multiply(mLocalOffset, mLocalRotation);
const mWorldArm = multiply(mGear, mLocalArm);

Meshless FEA WoS

The FEA example replaces a mesh with exact boundary distance queries. PGA line walls and a signed-distance disk feed a Walk on Spheres process that estimates the harmonic temperature field by jumping to the edge of the largest guaranteed-empty circle.

$$ \begin{aligned} d(P, \partial \Omega) &= \min \left(\min_i \left\| P \wedge L_i \right\|_{\infty}, \left|\sqrt{x^2 + y^2} - r\right|\right) \\ X_{k+1} &= X_k + d_k(\cos \theta_k, \sin \theta_k) \\ u(x) &= \mathbb{E}[g(X_{\tau})] \end{aligned} $$ 
const query = getShortestDistance(curr);
const angle = Math.random() * Math.PI * 2;

curr = point(
  getX(curr) + Math.cos(angle) * query.d,
  getY(curr) + Math.sin(angle) * query.d
);

Adding examples

The structure is intentionally small enough to grow.

New entries do not need a framework. They just need a demo route, a notes route, and one short place on the homepage. That keeps the library easy to browse and easy to extend.
