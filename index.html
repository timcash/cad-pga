PGA Split

In PGA, Euclidean content and ideal content live in the same algebra, which is why distance, direction, and incidence stay tightly coupled.

Rigid Motion
$$ \mathbf{M} = \mathbf{T}\mathbf{R}, \qquad X' = \mathbf{M} X \widetilde{\mathbf{M}} $$

Translation and rotation compose into one motor, so the same sandwich action can move points, lines, planes, and full tool geometry.

Mesh Cleanup

This example tracks a polygon boundary in PGA and shows that the signed net area remains fixed even when the gauge point moves away from the polygon plane.

$$ A = \frac{1}{2}\left\| o \vee \sum_i E_i \right\|_{\infty} $$
  • The red number is the sum of absolute cone facets, so it changes with the apex.
  • The green number is the ideal norm of the net signed plane sum, so it stays invariant.
  • The demo is a concrete way to see boundary-based magnitude instead of triangulating by hand.

CAD / CAM

CNC Kernel Simulator

This example turns spindle rotation and toolpath translation into one motor, then uses the resulting sweep to motivate how a PGA-native CAM kernel can describe cutting motion.

$$ \mathbf{M}(t) = \mathbf{T}(d_x(t))\,\mathbf{R}(\theta(t)) $$
  • The white endmill is transformed by one motor instead of separate matrix pipelines.
  • The red cut volume stands in for a swept volume, the core shape a CAM kernel would subtract.
  • The same framework extends cleanly toward multi-axis motion without Euler-angle singularities.
