CNC Kernel Simulator

One motor carries the spindle spin and the feed motion together.

The demo is intentionally simple, but the geometry is doing something important: the white tool is moved by one PGA motor, not by separate translation and rotation pipelines.

Verification note

This page is a direct rigid-motion demo, not one specific paper implementation.

The math here is grounded in the Bivector and "Look Ma, No Matrices" PGA references. The code is an intentionally small motor demo rather than a reproduction of a single CAM paper.

Core equations

The moving tool is a motor acting on points.

$$ \begin{aligned} P(x,y,z) &= \mathbf{e}_{123} + x\mathbf{e}_{032} + y\mathbf{e}_{013} + z\mathbf{e}_{021} \\ \ell_z &= \mathbf{e}_{12} \\ \mathbf{R}_z(\theta) &= \cos\!\left(\frac{\theta}{2}\right) + \sin\!\left(\frac{\theta}{2}\right)\ell_z \\ \mathbf{T}_x(d_x) &= 1 - \frac{1}{2} d_x \mathbf{e}_{01} \\ \mathbf{M}(t) &= \mathbf{T}_x(d_x(t))\,\mathbf{R}_z(\theta(t)) \end{aligned} $$

The rotor turns the spindle about the tool axis. The translator advances the tool through the stock. Their geometric product gives one rigid-body transform, and because the code builds motor = translator * rotor, the geometry is rotated first and then translated.

In code

The JavaScript stays close to the notation.

const point = (x, y, z) =>
  1e123 - x * 1e023 + y * 1e013 - z * 1e012;

const rotor =
  Math.cos(theta / 2) + Math.sin(theta / 2) * 1e12;

const translator = 1 - 0.5 * dx * 1e01;
const motor = translator * rotor;

const drillTransformed = drillBase.map((poly) =>
  poly.map((p) => motor >>> p)
);

The corrected point helper matters here: it keeps the tool body aligned with the z-axis, so the spindle rotation and the rendered geometry agree.

Swept volume

The red block is a simple stand-in for the cut sweep.

$$ \mathcal{S} = \bigcup_t \mathbf{M}(t)\,\mathcal{T}\,\widetilde{\mathbf{M}(t)} $$

That equation is the ideal geometric sweep. In the current demo, the red volume is still a simple box approximation rather than a full Boolean sweep, but it is placed to cover the tool radius along the full x-travel. 

const cutVolume = makeSolidBox(
  cutTravel + drillRadius,
  drillRadius,
  stockParams.d + 0.01
);

Interpretation

Why the representation is attractive for CAM.

$$ p'(t) = \mathbf{M}(t)\,p\,\widetilde{\mathbf{M}(t)} $$
  • Every tool vertex follows the same rigid motion rule.
  • The algebra composes rotation and translation without switching representations.
  • The same pattern scales naturally toward richer toolpaths and multi-axis motion.
